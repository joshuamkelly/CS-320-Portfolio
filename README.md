# CS-320-Portfolio

In order to ensure that the code is functioning and secure, it is essential to build with validation and testing integrated into the development process, as opposed to adding them as an afterthought. When building code, if it is intended to handle input, it is essential to assume that the code will be given unexpected input. Therefore, it is crucial to focus on testing and validation. Writing tests is an essential part of programming, as it allows me to verify that what I think is happening is actually happening. It is essential to be confident that if I alter something, I will be aware of it almost immediately. When it comes to interpreting the requirements of the program, it is essential to slow down and actually read them. Instead of building something that makes sense to me, it is crucial to interpret each requirement as a specific rule that the program is expected to implement. Therefore, breaking each requirement into small, specific actions is essential to avoid assumptions and to ensure that the finished product is actually what is expected. The approach to designing the program is to keep it as simple as possible. Therefore, it is essential to have each class have one specific responsibility and to avoid cross-cutting concerns. Additionally, it is crucial to consider testing as part of the design. Therefore, if something is difficult to test, it is an indication that it can be improved.
